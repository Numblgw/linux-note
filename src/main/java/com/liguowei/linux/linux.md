1) 虚拟机网络连接的三种形式

    1) 桥接模式
    
        虚拟机与本机在同一个网段，但是ip不相同，可以和该网段内的其他主机通讯。但可能会导致该网段的IP地址冲突，所以一般不用桥接模式。
        
    2) NAT模式（网络地址转换方式）建议选择
    
        linux可以访问外网，不会造成ip冲突，但是外网中的其他主机访问不到linux。
        
    3) 仅主机模式
    
        linux是一个独立的主机，不能访问外网。    
        
2) linux 的目录结构

    linux 的目录中有且只有一个根目录。linux的各个目录存放的内容是规划好的，不用乱放文件。linux 是以文件的形式管理设备的，所以 linux 系统中一切皆为文件。

    1) "/" 根目录，是最上层的目录。
    
    2) "/bin" 存放linux常用的指令
    
    3) "/home" 存放普通用户的主目录，在linux中每个用户都有一个自己的目录，一般以用户名命名。
    
    4) "/root" 超级用户（管理员）的主目录。
    
    5) "/sbin" 存放超级用户（管理员）用的指令
    
    6) "/dev" 管理设备，将硬件映射为文件来管理。
    
    7) "/etc" 所有的系统管理所需要的配置文件和子目录。     
    
    8) "/lib" 系统开机所需要的最基本的动态链接库，其作用类似于 windows 里的 dll 文件。几乎所有的应用程序都需要用到这些共享库。
    
    9) "/opt" 给主机额外安装软件所存放的目录，例如：安装oracle数据库。
    
    10) "/usr' 用户的很多应用程序和文件都放在该目录下，类似于 windows 中的 program files 目录。
    
    11) "/lost+found" 一般是空的，系统非法关机后存放一些文件。
    
    12) "/boot" 存放启动 linux 时使用的一些核心文件，包括一些连接文件和镜像文件。
    
    13) "/proc" 这个目录是一个虚拟目录，它是系统内存的映射，访问这个目录来获取系统信息。
    
    14) "/srv" service的缩写，该目录存放一些服务启动之后需要提取的数据。
    
    15) "/sys" 该目录中安装了2.6内核中新出现的一个文件系统 ysfs
        
    16) "/tmp" 临时目录，存放一些临时的文件。 
     
    17) "/media" linux系统会自动识别一些设备，例如U盘，光驱等，当识别后，linux会把识别的设备挂载到这个目录下。
    
    18) "/mnt" 让用户临时挂在别的文件系统，我们可以将外部的存储挂在在/mnt/上，然后进入该目录就可以查看里面的内容了。
    
    19) "/usr/local" 这是一个给主机额外安装软件所安装的目录（软件安装过后存放的目录），一般是通过编译源码的方式安装的程序。 
     
    20) "/var" 这个目录中存放着在不断扩充着的东西，习惯将经常被修改的文件存放到该目录下。例如：日志文件。 
    
    21) "/selinux" SELinux 是一个安全子系统，他能控制程序只能访问特定文件。

4) vi / vim 的使用

    1) 常用的三种模式
        
        1) 正常模式（默认模式）：
            
            命令行下查看文件直接进入该模式。默认模式下可以使用方向键移动光标，使用复制、粘贴、删除等快捷键处理文档。不可以输入字符，按 i 键进入编辑模式，按 : 或 / 进入末行模式。
                    
        2) 插入模式（编辑模式）：
        
            可以对文本进行编辑，输入 esc 进入正常模式。
        
        3) 命令行模式（末行模式）：
    
            :wq 写入存盘，也就是保存退出。
            
            :q 直接退出，不保存，如果已进行修改则会提示并退出失败。
            
            :q! 强制退出，无提示。
            
    2) vim 常用快捷键（正常模式下）
    
        1) 拷贝当前行 yy，拷贝当前向下 x 行 xyy，按 p 粘贴。
        
        2) 删除当前行 dd，删除当前向下 x 行 xdd。
        
        3) 在文件中查找某个单词，输入 “/被查找的字符 回车”，按 n 选中下一个。    
            
        4) 设置 / 取消文件的行号， :set nu / :set nonu
        
        5) 到文件最末行 G，到文件首行 gg。    
            
        6) 撤销上一步操作 u，类似于 windows 中 ctrl + z。
        
        7) 直接跳转到 x 行，必须先显示行号，然后 “x + shift + g”。    
            
5) 各种指令

    1) 关机&重启
    
        关机和重启前都要先使用 sync 指令将内存中的数据同步到磁盘。
    
        1) shutdown -h now 立即关机
        
        2) shutdown -h 1 一分钟后关机
        
        3) shutdown -r now 立即重启
        
        4) halt 关机
            
        5) reboot 重启
        
        6) sync 将内存中的数据同步到硬盘
            
    2) 用户相关指令
    
        1) 用户切换和注销。
        
            1) “su 用户名” 切换用户。
            
                权限高的用户切换到权限低的用户不需要输入密码，反之需要。
                
                当需要切换到原用户时只需要使用 exit 指令即可。
        
            2) logout 注销用户。在图形运行界面无效，在运行级别 3 下有效。
        
        2) 用户管理
        
            #### 用户概念：
        
            每个用户都属于一个组，也可以切换到其他组，root 用户有单独的组—— root 组。
            
            /home/ 目录下有各个创建目录对应的家目录，当用户登录时会自动进入自己的家目录。
        
            1) “useradd 用户名” 添加用户
            
                如果没有指定组，那么就会创建一个跟用户名相同的组，并将该用户添加到组中。
                
                也可以通过 -d 指定目录，不指定的话就创建家目录。
                
            2) “password 用户名” 修改用户密码
            
                使用 管理员用户才能执行该命令。
            
            3) “userdel 用户名” 删除用户
            
                会保留家目录，如果加上 -r 参数会同时删除该用户的家目录。工作中一般不会删除家目录。
                
            4) “id 用户名” 查询用户信息
            
                可以查到 uid gid 组    
                                
            5) “whoami” 查看当前用户    
            
            #### 用户组概念：
            
            类似于角色，系统可以对有共性的多个用户进行统一的管理。
            
            1) “groupadd 组名” 新增组
            
            2) “groupdel 组名” 删除组
            
            3) “usermod -g 组名 用户名” 修改用户的组名
            
            #### 用户和组相关的文件
            
            1) /etc/passwd 存放用户配置文件
            
            2) /etc/group 存放组配置文件
            
            3) /etc/shadow 存放口令配置文件（密码和登录信息），该文件是加密的。
            
    3) 实用指令
    
        #### 指令运行级别
        
        1) 级别分类一下这些级别都在 /etc/inittab 文件中指定。                    
        
            1) 级别0：关机
            
            2) 级别1：单用户 可以在单用户模式修改root密码，用于忘记root密码时使用。
            
            3) 级别2：多用户无网络
            
            4) 级别3：多用户有网络
            
            5) 级别4：保留，目前未启用
            
            6) 级别5：图形界面
            
            7) 级别6：重启

        2) 相关指令
        
            1) “init 【012356】” 切换运行级别

            2) “man 【命令或者配置文件】” 
            
                查看这个命令可以加的参数等等
            
                “help 命令” 自带的帮助指令

        3) 文件目录相关
        
            1) “pwd” 显示当前目录的绝对路径
            
            2) “ls 选项 目录或者文件” 
            
                不仅可以显示目录的详细信息，还可以显示文件的详细信息。

            3) “cd 路径” 切换目录
            
                “cd ~” 回到家目录
                
                “cd ..” 回到当前目录的上一级目录
                
                “cd 【/绝对路径】” 从根目录一级一级开始定位
                
                “cd 【相对路径】” 从当前工作目录开始定位
                
            4) “mkdir 选项 要创建的目录” 
            
                创建目录，-p 可以创建多级目录
            
            5) “rmdir 选项 要删除的空目录” 
            
                删除目录，只能删除空目录，如果要删除非空目录，可以使用 “rm -rf 目录”
            
            6) “touch 文件名” 
            
                创建一个空文件，也可以创建多个空文件，文件名用空格分隔。
            
            7) “cp 选项 source dest” 
            
                拷贝，source 和 dest 都可以是绝对路径或者相对路径
            
                -r 递归复制整个文件夹    
                
                cp 前面加 \ 强制覆盖，不提示。
            
            8) “rm 选项 要删除的文件或目录” 
                
                删除文件或目录
            
                -r 递归删除整个文件夹
                
                -f 强制删除不提示
                
            9) “mv oldNameFile newNameFile” 
            
                移动或者重命名，如果填写路径就是移动文件，如果填写文件名就是修改文件名。
                
            10) “cat 选项 要查看的文件”
            
                以只读的方式打开文件。
                
                -n 显示行号
                
                后加 | more 可以分页显示，按空格下一页
                
            11) “less 要查看的文件” 指令
            
                用来分屏查看文件内容他与 more 类似，但比 more 更强大，支持各种显示终端。less 在显示文件内容时并不是一次将整个文件加载之后才显示，而是根据显示需要加载的内容，对于显示大型文件具有较高的效率。
            
            12) “>" 输出重定向，会覆盖原文件
            
                “>>” 追加，向文件后面追加内容，不会覆盖原文件。
                
                ls -l > 文件名
                
                cat 文件1 > 文件2    
            
            13) “echo 选项 输出内容” 输出内容到控制台。
            
                输出环境变量 echo $PATH
                
                输出普通文本 echo xxxx
                
            14) “head 选项 文件名”
            
                显示文件的开头内容，默认显示开头10行，也可以使用 【head -n 5 文件名】 显示前 5 行
            
            15) “tail 选项 文件名” 
            
                显示文件尾部的内容，默认显示最后10行，也可以用 -n 指定行数，与 head 类似。
                
                -f 可以实时追踪该文档的所有更新
            
            16) “ln -s 原目录或文件 软连接文件（文件的快捷方式）” 软连接指令
            
                类似于给某个文件或目录创建一个快捷方式。
                
                使用快捷方式打开后 pwd 查看的目录没有变化，也就是当前目录不会发生改变。
            
                使用 rm 删除软连接文件
                
            17) “history” 指令 查看已经执行过的历史指令。    
                
        4) 时间日期类相关指令        
        
            1)  date 显示当前日期和时间
            
                +%y 显示当前年份
                
                +%m 显示当前月份
                
                +%d 显示当前是哪一天
                
                可以组合使用设置日期显示的格式
                
                “date -s 字符串时间” 设置系统时间
                
            2) “cal 选项” 查看日历指令。
            
                cal 2020 显示 2020年一整年的日历    
            
        5) 搜索查询相关指令    
        
            1) “find 搜索范围 选项”
            
                将递归查找指定目录和其子目录，将满足条件的显示于终端。不加范围就搜索当前目录和当前目录的子目录。
                
                -name 文件名  按照文件名查找，可以使用通配符。
                
                -user 用户名   按照文件的拥有者查找，查找该用户拥有的文件
                
                -size +10M/-10M/10M 查找文件大小 大于/小于/等于 10M的文件       
            
            2) “locate 搜索文件”
                
                locate 指令可以快速定位文件路径。locate 利用事先建立的系统中所有文件名及路径的 locate 数据库快速定位文件，无需遍历整个文件系统，查询速度较快。
                
                但管理员必须定期更新 locate 时刻，运行第一次前必须使用 updatedb 指令创建 locate 数据库。
                          
            3) “grep 选项 查找内容 源文件”
            
                可以使用管道符 |
                
                cat Hello.java | grep class 查找 Hello.java 文件中的 class 单词
                
                -n 显示行号
                
                -i 忽略大小写，默认区分大小写。
                
        6) 压缩和解压
        
            1) “gzip/gunzip” 该指令压缩完之后不会保留原文件，直接对原文件进行压缩。
            
                gzip 文件名    压缩文件
                
                gunzip 文件名.gz 解压缩文件    
            
            2) “zip/unzip” 
            
                “zip 选项 XXXX.zip 要压缩的内容” 压缩
                
                -r 递归压缩
                
                “unzip 选项 XXXX.zip”  解压
            
                -d目录 指定解压后的文件存放的目录
            
            3) “tar 选项 XXX.tar.gz 要打包的内容”
            
                -zcvf 压缩
                
                tar -zcvf myhome.tar.gz /home/  将 home 目录下所有文件压缩为 myhome.tar.gz
                
                -zxvf 解压
            
                tar -zxvf myhome.tar.gz -C /opt 将 myhome.tar.gz 解压到 /opt 目录下，指定解压到的目录时，目录要事先存在。
